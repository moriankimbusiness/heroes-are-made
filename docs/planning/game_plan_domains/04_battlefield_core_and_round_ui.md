# 전장, 코어, 라운드 UI

## 코어(수레) 규칙
- 모험단 수레를 전투의 코어로 사용한다.
- 코어는 맵 중앙에 배치한다.
- 전투 최우선 보호 대상은 코어다.
- 코어는 개별 체력(HP)과 피해 적용 API를 가진다.
- 코어 월드 오브젝트 상단에 체력바와 수치 표시를 항상 유지한다.
- 코어 체력 UI는 health_changed 시그널 기반으로만 갱신한다.
- 코어 충돌 범위는 사각형(RectangleShape2D)으로 유지하며 기본 크기는 96x96(32px 타일 기준 3x3)이다.
- 코어 시각 범위는 충돌 사각형 크기와 동일하게 유지한다.
- 히어로/적 이동은 코어 충돌 범위 내부 진입을 허용하지 않는다.

## 패배 조건
- 코어 체력이 0이 되면 패배한다.
- 또는 모든 히어로가 전투 불능(Down) 상태가 되면 패배한다.
- 기존 통과 횟수 기반 패배 규칙은 사용하지 않는다.

## 라운드 UI 및 속도 제어
- 상단 중앙 라운드 타이머 우측에 속도 제어 UI를 배치한다.
- 제공 버튼: 일시정지, 1x, 2x
- 일시정지 = 0x, 1x = 1.0, 2x = 2.0 배율 규칙을 사용한다.
- 기본 시작 속도는 항상 1x다.
- 속도 변경은 버튼 이벤트 기반으로 처리한다.

## 카메라 규칙
- 전투 카메라는 수동 이동/줌을 지원한다.
- 이동 입력: W, A, S, D
- 줌 입력: 마우스 휠
- 휠 업은 축소, 휠 다운은 확대로 통일한다.
- 일시정지(0x) 상태에서도 카메라 이동/줌 입력은 허용한다.
- 현 단계에서는 맵 확장보다 카메라 기본 동작 안정화를 우선한다.
